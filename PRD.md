**版本**：v0.3  
**日期**：2026-06-29  
**项目性质**：挑战杯揭榜挂帅参赛项目（恒顺醋业命题）  
**截止时间**：2026-07-15  
**交付物**：项目报告 + 网页端产品 Demo（PPT演示）

---

## 1. 项目背景

食醋行业在产品区域化定制方面面临三个核心痛点：

1. **经验依赖**：配方调整主要依赖老师傅经验，缺乏数据支撑
2. **文献孤岛**：国标、专利、学术文献分散，工程师难以快速调用
3. **感官与工艺脱节**：电子鼻/舌的理化检测数据未能有效指导配方决策

智醯（AcetiBot）通过 RAG（检索增强生成）架构，将感官数据、领域知识库与大语言模型连接，为配方工程师提供可追溯、可落地的配方改良建议。

---

## 2. 目标用户

| 用户类型 | 场景 | 核心诉求 |
|---------|------|---------|
| 食醋企业配方工程师 | 开发新产品或改良现有配方 | 快速获取有依据的工艺参数建议 |
| 食品科学研究人员 | 查阅文献、对比配方 | 高效检索领域文献，发现规律 |
| 竞赛评委（演示场景） | 评估项目创新性与可落地性 | 直观看到AI+食品科学的结合点 |

## 3. 核心功能需求

### 3.1 知识库管理

- **支持格式**：PDF（国标、文献、专利）、Excel（实验数据、配方表）、TXT/Markdown
- **自动元数据提取**：文档类型按文件夹自动识别（standards/patents/papers/consumer/sensor）
- **文档规模**：学术论文 800–1000 篇 + 其他资料约 500 份，总计约 1500 份

### 3.2 配方查询与生成

| 功能 | 说明 |
|------|------|
| 自然语言提问 | 用户输入如"设计适合江浙人口味的米醋配方" |
| 查询改写 | LLM 自动将口语化问题改写为检索优化查询 |
| 混合检索 | 语义检索 + BM25 关键词检索 + 元数据结构化过滤 |
| Rerank 重排序 | Cross-encoder 对 Top-K 结果重排，提升精度 |
| CoT 推理 | 启用 Qwen3 thinking 模式，减少幻觉，增强推理 |
| 配方建议生成 | 输出【配方建议】【工艺参数】【文献依据】三段式结果 |
| 来源溯源展示 | 每条建议标注来源文献，上标可点击查看详情 |

### 3.3 地域化口味定制

- 系统从自然语言中自动识别目标地域（非强制）
- 结合消费者偏好数据调整检索权重
- 输出配方时标注"适配地域"与"风味差异点"

### 3.4 感官数据接入（电子鼻/舌）

- 支持上传 CSV 格式的电子鼻/舌检测数据
- 将感官特征向量作为额外上下文注入查询
- 界面展示感官雷达图（酸/甜/苦/鲜/咸五维度）

### 3.5 垂直领域规范性审查

- 生成的配方建议自动对照 GB/T 18187 等国家标准
- 如建议参数超出标准范围，给出警示提示

## 5. RAG 优化策略（已确认）

### 5.1 检索增强模块（优先级排序）

| 模块 | 优先级 | 状态 | 说明 |
|------|--------|------|------|
| 表格提取优化（pdfplumber） | P0 | 待实现 | 国标数据全在表格，当前会丢失 |
| 查询改写（Query Rewriting） | P0 | 待实现 | 口语化问题 → 检索优化查询 |
| Qwen3 thinking 模式（CoT） | P0 | 待实现 | 1行代码，减少幻觉 |
| 混合检索（语义 + BM25） | P1 | 待实现 | 提升关键词精确匹配 |
| Rerank 重排序 | P1 | 待实现 | Cross-encoder 提升Top-K质量 |
| 图片 OCR（PaddleOCR） | P2 | 待定 | 仅处理有文字的图片 |
| 视觉理解（Qwen-VL） | P2 | 待定 | 仅对核心文献图表处理 |

### 5.2 多模态文档处理策略（分层）

| 文档类型 | 数量 | 处理策略 |
|---------|------|---------|
| 国标/专利（standards/patents） | ~500份 | 重点提取**文字+表格**，pdfplumber解析 |
| 学术论文（papers） | 800–1000篇 | 提取**文字+摘要**，跳过图片 |
| 核心精选论文（人工筛选） | 10–20篇 | 图片提取 + Qwen-VL视觉描述 |
| 消费者评价（consumer） | 若干Excel | 结构化入库 |
| 感官数据（sensor） | CSV | 结构化处理 + 雷达图展示 |

> **决策依据**：800-1000篇论文均做视觉处理成本过高（约4000-5000张图），采用分层策略：全量文字RAG + 精选图片理解。

## 6. 数据需求

### 6.1 知识库数据（目标规模）

| 数据类型 | 来源 | 目标数量 | 优先级 |
|---------|------|---------|--------|
| 国家标准 | GB/T 18187、GB/T 19777 等 | ~14份（已收集） | P0 ✅ |
| 企业公开专利 | 恒顺固态发酵、在线监测等 | 20–50份 | P0 |
| 学术文献 | Food Chemistry、知网等 | 800–1000篇 | P0 |
| 核心风味物质数据 | PubMed（川芎嗪、乙酸等） | 50–100篇 | P1 |
| 生产环境数据 | 镇江四季温湿度、微生物数据 | 若干 | P1 |
| 消费者评价数据 | 恒顺各平台电商评价 | 若干Excel | P1 |

### 6.2 感官数据格式（电子鼻/舌）

期望输入格式（CSV）：
```
sample_id, sensor_acid, sensor_sweet, sensor_bitter, sensor_umami, sensor_salt, label
S001,       4.2,         1.1,          0.3,            2.8,          1.5,         镇江香醋
```

## 9. 产品边界（不做什么）

- ❌ 不做实时生产线控制接口
- ❌ 不做用户账号体系
- ❌ 不做模型微调（使用 Prompt 优化替代）
- ❌ 不做移动端适配
- ❌ 不对所有论文图片做视觉理解（仅精选10-20篇）
- ❌ 不做自监督验证循环（latency过高，竞赛时间不足）
