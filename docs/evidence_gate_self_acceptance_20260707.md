# 证据门禁自验收记录（2026-07-07）

## 本轮范围

本轮只处理两个最高优先级事项：

1. 把“回答准入 / 拒答”从 Prompt 约束升级为程序规则。
2. 建立一套可重复运行的本地验收题库。

## 已完成改动

### 1. 程序化证据门禁

- 已在 `query.py` 中加入：
  - `evaluate_evidence_gate`
  - `build_refusal_answer`
  - `build_gate_analysis_card`
- 已在普通查询与流式查询中接入门禁：
  - 门禁通过：允许调用大模型生成答案
  - 门禁未通过：直接返回拒答说明，不再继续生成

### 2. 门禁规则首版

已覆盖以下约束：

- `consumer_insight`
  - 缺少消费者子库证据时拒答
  - 统计/排序题缺少消费者表格证据时拒答
- `standard_compliance`
  - 缺少标准证据时拒答
- `patent_process`
  - 缺少专利证据时拒答
- `flavor_mechanism`
  - 缺少风味/机理论文证据时拒答
  - 指标问题未命中关键指标词时拒答
- `formula_process`
  - 缺少主证据层时拒答
  - 提到合规但未命中标准约束时拒答
  - 提到消费者偏好但未命中消费者辅助证据时拒答

### 3. 本地验收骨架

- 新增验收题库：
  - `docs/acceptance_cases_v1.json`
- 新增验收脚本：
  - `eval_acceptance.py`

该脚本会检查：

- 路由是否符合预期
- 门禁是否符合预期
- Top1 证据来源是否基本合理
- 预留 `paper` 子库 case 是否保持可扩展

## 当前仍保留的已知风险

本轮通过的是“是否允许回答”的门禁，不等于所有“允许回答的问题”都已经拿到最优首证据。

仍存在的风险：

1. 某些 `flavor` 论文的 `table_fact` 仍有噪声，尤其是复杂微生物-风味相关矩阵类表格。
2. `哪组样品总酸更高？` 这类问题在当前代表性小库下仍会先召回噪声风味表格，但已会被门禁拒答。
3. 老旧扫描专利全量重建成本仍然高，不适合现在就做正式全库更新。

## 2026-07-07 补充进展

- `formula_process_layered_answer`
  - Top1 已从 `table_fact` 变为 `paragraph`
