# 多模态 RAG 一期自验收记录（2026-07-07）

## 本轮目标

1. 压制 PDF 中双栏正文被误识别为表格的问题。
2. 让数值/比较类问题优先命中 `table_fact` / `table_summary`。
3. 对配方工艺问题使用分层证据，不再把所有子库混在一起。

## 本轮实现

### 解析侧

- `ingest.py`
  - 增加表格结构质量特征：
    - `header_keyword_count`
    - `header_sentence_like_count`
    - `header_max_length`
    - `header_avg_length`
    - `sentence_like_ratio`
    - `long_cell_ratio`
    - `numeric_like_ratio`
    - `multiline_cell_ratio`
  - 裁掉表头前噪声行，避免整页前言混入表格正文。
  - 对疑似双栏正文/长段落误判表格直接降级为 `review` / `blocked`。
  - 同页存在更结构化候选时，默认压制 `text` 策略抽出的伪表格。
  - 对“整列数据挤进单个单元格”的表格，不再自动生成 `table_fact`。

### 检索侧

- `query.py`
  - 为数值/比较类问题增加表格证据优先权重。
  - 为结果增加关键词命中、数值型偏好、关键指标缺失惩罚。
  - 修正路由：
    - `哪组样品总酸更高` 不再误进 `standard_compliance`
    - 配方/工艺建议题继续走 `formula_process`
  - `formula_process` 继续维持分层证据：
    - 工艺与风味论文
    - 专利工艺
    - 标准约束
    - 消费者偏好

### 问题验证

#### 1. `消费者评价里最关注哪些参数？`

- 结果：通过
- 前 3 条均命中 `parameter_statistics.csv` 的 `table_fact`
- 排序结果与数值一致：
  - 醋味/口感/味道
  - 包装/物流
  - 价格/性价比

#### 2. `哪组样品总酸更高？`

- 结果：未完全通过
- 已改进：
  - 路由已经从 `standard_compliance` 修正为 `flavor_mechanism`
- 仍存在问题：
  - 当前首屏仍可能被噪声较高的风味论文表格事实占据
  - 说明“关键指标词命中 + 表格降噪”还需要继续加强

#### 3. `如何优化配方让酸香更协调，同时兼顾合规和消费者接受度？`

- 结果：部分通过
- 已实现：
  - 返回结果中已包含专利、标准、消费者偏好三个证据层
  - 不再把问题误判成单一消费者或标准问题
- 仍存在问题：
  - 风味论文主证据的前排结果里仍可能混入噪声 `table_fact`
  - 说明“建议题优先正文段落”的策略还需要再加强

## 结论

### 已确认可用

- 消费者统计类 CSV -> 可稳定生成可引用事实块
- 标准与专利 -> 路由与分层证据框架已可用
- 双栏正文误判 -> 相比之前已明显收敛
- `formula_process` 建议题 -> 首屏主证据已从噪声 `table_fact` 收敛为正文段落，更符合“先可解释，再生成建议”的目标

### 暂不建议直接宣称完全解决

- 部分风味论文中的复杂表格仍会生成噪声较高的事实块
- 全量老旧扫描专利很多，OCR 成本高且不少文件本身无法稳定解析
- 因此当前更适合继续本地迭代后，再做正式全量重建

## 下一步建议

1. 对高噪声风味论文增加“表格事实白名单/黑名单”或更严格的页级门禁。
2. 对 `formula_process` 建议题进一步提高正文段落优先级。
3. 对标准大文件如 `GB2760-2024kw.pdf` 增加更细的分片与去重策略，避免一次产生数千块。

## 2026-07-07 补充进展

在继续收 `flavor` 高噪声表格后，已新增以下抑制策略：

- 对“表头碎片化短语 + 前几行混有表题/图题”的候选表格，降为 `review`
- 对这类候选表格不再生成 `table_fact`
- `formula_process` 建议题在组内选择时，优先保留正文段落顺序
