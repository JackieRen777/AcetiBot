# 多模态 RAG 一期计划

## 目标

一期不追求“一次性吃下所有多模态”，而是优先解决当前最影响回答准确性的环节：

1. 论文和报告中的表格能够被单独提取、验收、引用。
2. 表格中的关键数值能够变成可检索的事实块，而不只是附在正文里。
3. 解析结果要有中间产物，能够人工抽查，不接受“看起来入库了但不知道有没有乱码”。

## 一期范围

### 覆盖文件类型

- PDF
- Excel (`.xlsx` / `.xls`)
- CSV / TSV
- TXT / Markdown

### 一期重点

- 表格优先解析
- 表格结构化产物落盘
- 表格事实块生成
- 解析验收指标补强

### 一期暂不解决

- 图片图表中的曲线数值重建
- CAJ 原生解析
- 复杂扫描件的高精度版面还原
- 图像区域级别引用

## 证据分层

### 文档层

- `standard`：约束层，用于违禁/限量/标签/用量边界
- `paper`：工艺主证据层
- `flavor`：机理解释层
- `patent`：产业实现层
- `consumer`：偏好校正层

### 证据块层

每个文档不再只拆成“纯正文 chunk”，而是拆成以下证据块：

- `paragraph`：正文段落
- `table_summary`：整表摘要
- `table_row`：表格行级内容
- `table_fact`：由表格归纳出的事实句
- `table_preview`：人工抽查用的 Markdown / JSON 产物

## 解析策略

### 1. 文档识别

- 判断文件类型：PDF / Excel / CSV / TSV / Text
- 判断页面类型：text / mixed / image / scanned
- 对 PDF 额外识别是否存在可抽取表格

### 2. 表格提取

对 PDF / Excel / CSV 统一输出：

- `tables.json`
  - 保留表格原始二维结构
  - 记录页码、表格编号、行列数、非空率
- `table_facts.jsonl`
  - 将关键表格内容转换为可检索事实句
  - 每条事实带页码、表格编号、行号、字段名

### 3. 表格质量门禁

每张表都要打质量标签：

- `pass`：结构基本完整，可用于入库
- `review`：表格存在空列、错位、全空头等问题，保留产物但不自动转为事实块
- `blocked`：严重错位或几乎空白，不用于事实抽取

判定重点：

- 非空单元格占比
- 头部有效性
- 行列一致性
- 是否只有噪音字符

### 4. 入库策略

- 正文 chunk 继续入库
- 表格摘要 chunk 入库
- 表格行级 chunk 入库
- 表格事实块优先入库

这样问“参数是多少”“哪组最高”“显著差异在哪”时，优先命中表格证据，而不是模糊命中正文讨论。

## 产物设计

每个 `parsed_docs/<doc>/` 下新增：

- `combined.md`
- `manifest.json`
- `pages/page_xxx.md`
- `tables.json`
- `table_facts.jsonl`

`manifest.json` 要补充：

- `table_count`
- `table_fact_count`
- `table_quality_summary`

## 一期自验收流程

### A. 解析级验收

对每次新增资料至少抽查 3 类样本：

1. 有规范表格的论文 PDF
2. 评论/统计类 CSV
3. 有复杂版式但含表格的 PDF

检查项：

- `tables.json` 是否真实存在
- 表头是否不是全空
- 行列数是否合理
- `table_facts.jsonl` 是否生成了可读事实句
- `parse_report.csv` 是否能反映表格质量

### B. 检索级验收

至少验证 3 类问题：

1. 数值型：如“消费者关注度最高的参数是什么”
2. 对比型：如“哪组样品总酸更高”
3. 条件型：如“什么工艺条件下某指标最高”

通过标准：

- 来源命中表格相关 chunk
- 引用片段能看到数值/条件，而不是只有模糊正文

### C. 风险记录

如果出现以下情况，一期先记录为“已识别风险”，不强行伪解决：

- `pdfplumber` 抽到全空表
- 合并单元格导致表头错位
- 图表数值无法从图像恢复
- 扫描件 OCR 可读但表格结构不可恢复

## 本轮执行目标

本轮先落 3 件事：

1. 在 `ingest.py` 中加入表格结构化产物输出能力
2. 将表格事实块纳入入库 chunk
3. 用本地样本文档跑一轮自验收，确认产物可用
